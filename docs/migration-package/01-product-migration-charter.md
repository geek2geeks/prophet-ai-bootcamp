# Product Migration Charter

## Purpose

Define what the rebuild is trying to preserve, what must change, and what success looks like for a modern web product.

## Key sections

- Product intent and non-goals
- Why rebuild now
- Success criteria for the Next.js platform
- Content preservation principles
- UX reinvention principles
- MVP scope for the rebuilt product
- Out-of-scope items for phase 1
- Risks, assumptions, and decisions needing owner input

## How to use the current repo as source material

- Use `lib/course.py` to identify the core educational promise, day-by-day progression, point system, and Prophet Lite narrative that must survive the migration.
- Use `lib/ai.py` to capture the current product philosophy: actuaries becoming AI-native builders, spec-driven workflows, and the role of the tutor.
- Use `prophet_reference_vida.md` to anchor the wedge, target user, and incumbent comparison.
- Use `CURRICULUM_REVIEW.md` and `docs/deepseek-v3.2-assessment.md` to carry forward current critiques and realistic AI constraints.
- Explicitly state that Streamlit page structure is not part of the preservation contract.
